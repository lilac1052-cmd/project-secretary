## 한 줄 요약
여러 프로젝트를 혼자 동시에 관리하는 사람이, 캘린더·메신저·구글문서·노션에 흩어져 있던 일정과
계획을 눈으로 대조하던 방식을, 한곳에 모아 마감을 자동으로 챙겨주는 대시보드로 바꿨다.

## 자동화 전 — 원래 어떻게 했나
- **하던 일**: 새 프로젝트가 시작되면 캘린더에 일정을 적고, 카카오톡으로 받은 요청사항을
  따로 메모하고, 관련 자료는 구글문서·노션 페이지에 흩어져 저장했다. 여러 프로젝트를 동시에
  진행할 땐 이 여러 곳을 일일이 열어보며 "지금 뭐가 급한지"를 머릿속으로 종합해야 했다.
- **불편했던 점**: 종합적으로 한눈에 볼 수 있는 곳이 없어, 적어두고도 마감을 놓치는 일이 있었다.
  프로젝트 수가 늘수록 어디에 뭘 적었는지부터 다시 찾아야 했다.

## 무엇을 자동화했나
- **앱이 대신하는 일**: 프로젝트를 한 번 등록하면(이름·유형·파트너·기간·핵심요건·단계·할일),
  대시보드가 마감이 임박했거나 지연된 프로젝트를 자동으로 골라 맨 위에 보여준다. 모든 프로젝트의
  단계·할일 마감을 가까운 순으로 모아 보여주는 목록과, 달력 위에 점으로 찍어 보여주는 화면도
  자동으로 만들어진다. 프로젝트 기획 문서를 파일로 올리면 Gemini가 내용을 읽어 이름·유형·
  파트너·기간·핵심요건·단계까지 입력란에 자동으로 채워준다.
- **일부러 사람한테 남긴 일**: 단계 상태 변경과 할일 체크는 자동으로 판단하지 않고 사람이 직접
  클릭해서 갱신한다 — 진행 상황 판단은 본인만 정확히 알 수 있는 영역이라 자동 추정하지 않았다.
  자동 채우기로 채워진 값도 등록 전에 검토·수정할 수 있게 열어 두었다 — 문서 해석이 틀릴 수
  있기 때문이다.

## 사람이 개입하는 지점 (Human-in-the-Loop)
- 문서 자동 채우기 결과는 바로 저장되지 않고 폼에 채워지기만 한다. 사용자가 눈으로 확인하고
  고친 뒤 직접 "등록하기"를 눌러야 실제로 저장된다 — LLM이 날짜나 담당자를 잘못 읽어도 그대로
  저장되는 사고를 막기 위함이다.
- 단계 완료·할일 체크는 시스템이 추정하지 않고 항상 본인이 클릭해서 바꾼다.

## 업무 흐름 — 전 → 후
- **전**: 캘린더·메신저·구글문서·노션을 각각 열어보며 머릿속으로 "지금 뭐가 급한지" 종합했다.
- **후**: 프로젝트 비서 대시보드 하나만 열면 임박·지연 프로젝트가 맨 위에 보이고, 문서 한 장만
  올리면 새 프로젝트 등록의 절반 이상이 자동으로 채워진다.

  그대로 재사용할 수 있어 편하다.
