# 프로젝트 비서

기술 이름(slug): `project-secretary`

## 한 줄 소개
여러 프로젝트의 세부구성·핵심 요건·진행과정·마감을 한눈에 보여주는 개인 비서형 대시보드

## 핵심 흐름
1. 프로젝트를 등록하고 초반 계획(유형·파트너·기간·핵심요건·단계)을 적어둔다
2. 프로젝트 탭을 열어 현재 진도·다음 마감·해야 할 일을 확인하거나, 마감 모아보기에서 전체 일정을 본다
3. 여러 프로젝트를 동시에 추적해도 헤매지 않는다 — 중요한 타이밍을 놓치지 않는다

## 참고 앱/사이트
Notion — 프로젝트별 페이지·진행 상태 보드 형태가 마음에 들었음

## 설정
- 로그인: 필요 없음 (혼자만 씀, 그냥 접속)
- LLM API: 불필요 (정해진 데이터를 입력·저장·표시·집계하는 게 핵심)
- 외부 연동: 없음 (문서 파일은 업로드가 아니라 외부 링크 텍스트로만 연결)
- 민감정보: 없음

## 화면
1. **대시보드(홈)** — `/`
   - 보임: 1순위 마감이 임박했거나 지연된 프로젝트 카드(경고 표시) → 2순위 나머지 프로젝트 카드(이름·진행 단계·다음 마감일) → 3순위 "새 프로젝트" 버튼
   - 동작: 프로젝트 카드를 누르면 상세로 이동
   - 데이터: 읽음: projects, stages, tasks
   - 상태: 로딩 스피너 / 프로젝트 0개일 때 "첫 프로젝트를 등록해보세요" 안내 / 오류 시 재시도 메시지
   - 디자인: design/dashboard.html
2. **마감 모아보기** — `/deadlines`
   - 보임: 1순위 가장 가까운 마감 순으로 정렬된 목록(단계 마감 + 할일 마감 통합, 프로젝트명 표시) → 2순위 지난 마감(지연) 구분 표시
   - 동작: 항목을 누르면 해당 프로젝트 상세로 이동
   - 데이터: 읽음: stages, tasks, projects
   - 상태: 로딩 / 다가올 마감 없을 때 "예정된 마감이 없습니다" / 오류 메시지
   - 디자인: design/deadlines.html
3. **프로젝트 상세** — `/projects/[id]`
   - 보임: 1순위 프로젝트 이름·유형·파트너·기간·핵심요건 → 2순위 단계 목록(상태별 색상) → 3순위 할일 목록(체크박스)
   - 동작: 단계 상태 변경, 할일 체크/추가, 문서 링크 클릭
   - 데이터: 읽음/씀: projects, stages, tasks
   - 상태: 로딩 / 단계·할일 없을 때 "아직 등록된 단계가 없습니다" / 저장 중·성공·실패 피드백
   - 디자인: design/project-detail.html
4. **새 프로젝트 등록** — `/projects/new`
   - 보임: 1순위 이름·유형·파트너 입력칸 → 2순위 기간(시작일~마감일)·핵심요건·문서 링크 → 3순위 단계 목록 입력(여러 개 추가 가능)
   - 동작: 단계를 여러 개 추가/삭제하며 입력 후 "등록" 버튼
   - 데이터: 씀: projects, stages
   - 상태: 제출 중 스피너 / 성공 시 상세로 이동 / 실패 시 오류 메시지
   - 디자인: design/project-new.html
5. **캘린더 보기** — `/calendar`
   - 보임: 1순위 월간 캘린더 그리드, 날짜 칸마다 그날 마감인 단계·할일을 프로젝트별 색상 점/라벨로 표시 → 2순위 날짜를 누르면 그날 마감 목록을 옆이나 아래에 펼쳐서 보여줌
   - 동작: 월 이동(이전/다음), 날짜 클릭해 상세 목록 보기, 항목 클릭 시 해당 프로젝트 상세로 이동
   - 데이터: 읽음: projects, stages, tasks
   - 상태: 로딩 / 그 달에 마감이 없을 때 "이 달엔 마감이 없습니다" / 오류 메시지
   - 디자인: design/calendar.html

## 데이터 (Supabase 테이블)
- **projects**
  - `id` uuid — 기본 키
  - `name` text — 프로젝트 이름
  - `type` text — 프로젝트 유형
  - `partner` text — 파트너/단체
  - `start_date` date — 시작일
  - `end_date` date — 마감일
  - `key_requirements` text — 핵심 요건
  - `doc_link` text — 관련 문서 링크
  - `created_at` timestamptz — 생성일
  - 관계: 없음
- **stages**
  - `id` uuid — 기본 키
  - `project_id` uuid — 소속 프로젝트
  - `name` text — 단계 이름(예: 계획/진행/완료)
  - `status` text — 계획중/진행중/완료
  - `due_date` date — 단계 마감일
  - `order_index` int — 표시 순서
  - 관계: project_id → projects.id
- **tasks**
  - `id` uuid — 기본 키
  - `project_id` uuid — 소속 프로젝트
  - `stage_id` uuid — 소속 단계(없을 수 있음)
  - `title` text — 할일 내용
  - `done` boolean — 완료 여부
  - `due_date` date — 할일 마감일(없을 수 있음)
  - 관계: project_id → projects.id, stage_id → stages.id
